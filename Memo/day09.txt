기본생성자
	클래스()
	생성자가 명시적으로 정의되어있지 않다면 컴파일러가 자동으로 기본 생성자를
	추가해준다

초기화 생성자
	클래스명(매개변수,...)
	메소드처럼 값을 매개변수로 전달받아 인스턴스 변수를 초기화하는데에
	사용하는 생성자

클래스 내에 생성자가 명시적으로 정의되어있는 경우
컴파일러는 기본 생성자를 자동으로 추가해주지 않는다
따라서 여러개의 생성자를 사용하고 싶다면 생성자 오버로딩을 통해
사용할 수 있다


static(제어자, modifier)
	static 변수 : 모든 객체(인스턴스)가 공유하는 변수
	static 메소드 : 모든 객체(인스턴스)에 동일하게 동작하는 메소드

	클래스명으로 바로 사용이 가능하고, 클래스명으로 바로 사용하는 것이
	바람직하다

접근제어자(access modifier)
			같은클래스	같은패키지	다른패키지	
					다른클래스	다른클래스
	private		O		X		X
	default		O		O		X
	protected		O		O		X(자식만가능)
	public		O		O		O


private 변수를 사용하는 경우
public으로 설정된 메소드를 통해서 우회적으로
변수에 값을 저장하고,
변수에 저장된 값을 가져와야한다

이러한 기능을 수행하는 메소드를 게터와 세터라고 이야기하며
통상적으로 set변수명() 혹은 get변수명() 으로 작명한다

private 변수를 사용하는 이유
	프로그램상 비정상적인 값을 메소드를 통해 한 번 걸러줌으로써
	안정성을 높일 수 있다.
